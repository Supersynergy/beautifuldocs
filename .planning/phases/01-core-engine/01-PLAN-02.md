# Plan 02: Markdown Parser & Frontmatter

**Phase:** 1 - Core Engine
**Goal:** Parse Markdown files with frontmatter configuration

## Steps

1. **Create `src/lib/parser.ts`**:
   - Use `gray-matter` to parse frontmatter
   - Use `marked` to convert Markdown to HTML
   - Support GitHub-flavored Markdown (tables, code blocks, strikethrough)
   - Return structured data: { frontmatter, html, raw }

2. **Define frontmatter schema**:
   - `title`: document title
   - `template`: template name (editorial, dark, minimal, neon)
   - `format`: 'a4' | '16:9'
   - `palette`: color palette name or custom colors

3. **Create `src/lib/types.ts`** with TypeScript interfaces

4. **Test parser** with sample markdown file

## Success Criteria
- Parser correctly extracts frontmatter from test files
- Markdown converts to valid HTML with tables and code blocks
- TypeScript types are complete
